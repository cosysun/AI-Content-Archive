## 一、为什么我要把公众号交给 OpenClaw 运营

说实话，我做公众号快三年了，最痛苦的不是写文章本身，而是**写文章之前和写完之后**的那些破事儿。

选题要刷半小时热搜，写完要对着屏幕改标题改到怀疑人生，排版要手动加粗加链接，发出去还要担心有没有错别字、有没有违禁词……每篇文章真正"写"的时间也就两小时，但前后折腾加起来能干掉一整个下午。

去年底开始用 OpenClaw 之后，这个流程被拆解重组了一遍。现在的节奏大概是：

- **早上 7 点**：OpenClaw 自动跑热点监控，生成今日选题推荐
- **早上 8 点**：我看一眼手机，选一个选题，回复个数字
- **上午 10 点**：文章初稿生成完毕，推送到草稿箱
- **下午 2 点**：我花 15 分钟做最终确认，点发布

整个过程我主动参与的时间不超过 20 分钟，其他的都是 OpenClaw 在跑。

这篇文章就是把这套流程完整拆解一遍，包括怎么配置、怎么调教、踩过哪些坑。

## 二、环境搭建：10 分钟把 OpenClaw 跑起来

### 2.1 你需要准备什么

在开始之前，先把这几样东西备齐，后面会反复用到：

| 材料 | 用途 | 获取方式 |
|------|------|----------|
| 阿里云账号（已实名） | 部署服务器 | aliyun.com 注册 |
| 百炼 API-Key | 驱动 AI 写作能力 | 阿里云百炼控制台创建 |
| 微信公众号 AppID + AppSecret | 对接公众号草稿箱 | 公众号后台「开发者ID」 |
| 服务器公网 IP | 微信回调配置 | 购买服务器后获得 |

> **注意**：公众号必须是**已认证的订阅号或服务号**，未认证账号没有开发者权限，无法调用草稿箱 API。

### 2.2 服务器选购建议

用阿里云轻量应用服务器，直接选 OpenClaw 专属镜像（应用镜像里搜"OpenClaw"或"Clawdbot"）。

配置别太抠：
- **地域**：选中国香港（免备案，微信 API 访问正常）
- **规格**：2 核 2GB 内存起步，同时跑多个自动化任务建议 4 核 4GB
- **带宽**：5Mbps 够用，如果要跑图片生成可以适当加大

购买完成后，进入实例详情，把公网 IP 记下来，后面配置微信回调要用。

### 2.3 端口放通（必做，很多人卡在这里）

进入 WebShell，执行：

```bash
# 放通 OpenClaw 核心通信端口
firewall-cmd --add-port=18789/tcp --permanent

# 放通微信公众号回调端口
firewall-cmd --add-port=80/tcp --permanent

# 重载防火墙
firewall-cmd --reload

# 验证，看到这两个端口就对了
firewall-cmd --list-ports
```

同时在阿里云控制台的「防火墙」里，也要手动添加 18789 和 80 端口的入站规则，**两边都要放通，缺一不可**。

### 2.4 配置百炼 API-Key

```bash
# 进入 OpenClaw 配置目录
cd /root/.openclaw 2>/dev/null || cd /root/.clawdbot

# 写入 API Key（替换成你自己的）
cat > config.yaml << EOF
llm:
  provider: aliyun-bailian
  api_key: "你的百炼API-Key"
  model: qwen-max
  temperature: 0.7
EOF

# 重启服务
systemctl restart openclaw
```

配置完成后，访问 `http://你的服务器IP:18789`，能看到对话界面就说明跑起来了。

## 三、对接微信公众号：让 OpenClaw 直接推送草稿

这一步是整个流程的核心，也是坑最多的地方。

### 3.1 公众号后台配置

登录微信公众号后台，进入「设置与开发」→「基本配置」：

1. 开启「服务器配置」
2. **URL** 填：`http://你的服务器IP/wechat/callback`
3. **Token** 自定义一个字符串，记下来
4. 消息加解密方式选「明文模式」（调试阶段用，稳定后可改安全模式）
5. 点「提交」，微信会向你填的 URL 发一个验证请求

如果验证失败，99% 是 80 端口没放通，回去检查防火墙。

### 3.2 在 OpenClaw 里配置公众号插件

> **注意**：以下插件指令以 OpenClaw 官方文档为准，不同版本的指令格式可能有所差异，建议先查阅 [官方文档](https://openclaw.ai/docs) 确认。

在 OpenClaw 对话框里发送：

```
/plugin install wechat-mp
```

安装完成后配置凭证：

```
/config wechat-mp set appid 你的AppID
/config wechat-mp set appsecret 你的AppSecret
/config wechat-mp set token 你在公众号后台设置的Token
```

验证连接是否正常：

```
/plugin test wechat-mp
```

返回 `连接成功，账号：你的公众号名称` 就对了。

### 3.3 测试推送草稿

发一条指令测试：

```
帮我写一篇关于"AI工具提升效率"的公众号文章，500字，写完同步到草稿箱
```

等两三分钟，去公众号后台的「素材管理」→「草稿箱」看，文章出现了就说明整条链路通了。

## 四、自动化选题：每天早上 7 点的热点简报

手动选题是最浪费时间的环节之一。我用 AIContentFlow 的监控脚本配合 OpenClaw 做了一套自动化选题流程。

### 4.1 搭建热点监控脚本

如果你已经有 AIContentFlow 或类似的监控工具，直接对接就行。如果没有，可以用下面这个简化版：

```python
# hot_topics.py - 简化版热点监控
import requests
import json
from datetime import datetime

def fetch_hackernews_top():
    """抓取 HackerNews Top Stories"""
    url = "https://hacker-news.firebaseio.com/v0/topstories.json"
    ids = requests.get(url, timeout=10).json()[:20]
    
    stories = []
    for story_id in ids:
        story = requests.get(
            f"https://hacker-news.firebaseio.com/v0/item/{story_id}.json",
            timeout=5
        ).json()
        if story and story.get('score', 0) > 200:
            stories.append({
                'title': story.get('title', ''),
                'score': story.get('score', 0),
                'url': story.get('url', '')
            })
    return stories

def generate_report(stories):
    """生成选题报告"""
    today = datetime.now().strftime('%Y-%m-%d')
    report = f"# 热点选题报告 {today}\n\n"
    
    for i, s in enumerate(stories[:5], 1):
        report += f"## 选题{i}：{s['title']}\n"
        report += f"- 热度：{s['score']}\n"
        report += f"- 链接：{s['url']}\n\n"
    
    return report

if __name__ == '__main__':
    stories = fetch_hackernews_top()
    report = generate_report(stories)
    
    with open('/tmp/hot_topics.md', 'w') as f:
        f.write(report)
    
    print(report)
```

### 4.2 配置定时任务

```bash
# 每天早上 7 点执行热点监控
crontab -e

# 添加这一行
0 7 * * * python3 /root/hot_topics.py && curl -X POST http://localhost:18789/api/notify -d '{"message":"今日热点已生成，请选题"}'
```

### 4.3 让 OpenClaw 自动生成选题建议

热点数据拿到后，发给 OpenClaw 做二次分析：

```
根据以下热点数据，结合我的公众号定位（AI工具、效率提升、技术科普），
推荐3个最适合创作的选题，每个给出：
1. 推荐标题（吸引人的中文标题）
2. 核心角度（我的公众号读者会感兴趣的切入点）
3. 预计阅读量（高/中/低）

热点数据：[粘贴hot_topics.md的内容]
```

OpenClaw 会结合你的历史创作风格和当前热点，给出有针对性的选题建议，比自己盯着热搜猜靠谱得多。

## 五、AI 写作：从选题到初稿的完整提示词工程

选好题之后，写作阶段的关键是**提示词设计**。一个好的提示词能让初稿质量直接提升一个档次，省掉 60% 的后期修改工作。

### 5.1 基础写作指令模板

```
你是一个专注于AI工具领域的公众号作者，风格是：专业但不装、实用优先、喜欢用真实案例说话。

请写一篇公众号文章，要求如下：

【选题】：{选题标题}
【字数】：3000-4000字
【结构】：
- 开头：用一个真实场景或痛点引入（不要用"在当今时代"这种废话开头）
- 正文：3-4个核心章节，每章节有小标题
- 结尾：给读者一个可以立刻执行的行动建议

【风格要求】：
- 多用短句，少用长难句
- 每隔3-4段加一个小结或金句
- 避免"综上所述""不难发现"等AI腔调词汇
- 数据要有来源，不要捏造

【特殊要求】：
- 加入2-3个具体的操作步骤或代码示例
- 结尾设计一个互动引导（如"你在用什么工具做XX？评论区聊聊"）
```

### 5.2 分段写作法（适合长文）

对于 5000 字以上的长文，不要一次性让 OpenClaw 生成全文，容易出现后半段质量下滑的问题。建议分段来：

**第一步：生成大纲**
```
请为《{文章标题}》生成一个详细大纲，包含：
- 5-7个主要章节
- 每章节的核心论点（1-2句话）
- 预计字数分配
```

**第二步：逐章写作**
```
请根据大纲，写第{X}章：{章节标题}
要求：{字数}字，延续前面章节的风格，
前文摘要：{前文关键信息}
```

**第三步：整合润色**
```
请将以下各章节整合为完整文章，
检查：逻辑连贯性、风格统一性、段落过渡是否自然
[粘贴各章节内容]
```

### 5.3 实战提示词：公众号爆款结构

根据我这一年多的数据，以下结构在公众号上完读率最高：

```
写一篇公众号文章，使用以下结构：

**开头钩子**（150字以内）：
用一个让读者感同身受的场景或问题，比如"你有没有过这种感觉……"

**核心价值承诺**（100字）：
明确告诉读者，读完这篇文章能得到什么

**主体内容**（分3个章节，每章800-1000字）：
- 章节1：问题/现状分析
- 章节2：解决方案/工具介绍（含步骤）
- 章节3：进阶用法/注意事项

**行动号召**（200字）：
给一个具体可操作的建议，引导评论互动
```

## 六、降AI味：让文章读起来像人写的

这是整个流程里**最需要人工参与**的环节。AI写的稿子，AI味往往集中在这几个地方：

### 6.1 AI味的典型特征（对照自查）

**词汇层面：**
- ❌ "不难发现" → ✅ "你会发现"
- ❌ "综上所述" → ✅ 直接写结论，不需要这个词
- ❌ "毋庸置疑" → ✅ "这一点很确定"
- ❌ "在当今社会" → ✅ 直接切入主题
- ❌ "值得注意的是" → ✅ 直接说那件事

**句式层面：**
- ❌ 每段都是"首先...其次...最后..." → ✅ 用更自然的过渡
- ❌ 每句话都是完整的主谓宾结构 → ✅ 适当用短句、省略句
- ❌ 段落长度高度均匀 → ✅ 长短段落交错

**内容层面：**
- ❌ 观点正确但空洞 → ✅ 加入具体案例、数据、个人体验
- ❌ 没有立场 → ✅ 表达明确的观点，甚至有点偏激也没关系

### 6.2 用 OpenClaw 做降AI味润色

把初稿交给 OpenClaw，用这个提示词做润色：

```
请对以下文章进行降AI味润色，具体要求：

1. 找出所有"AI腔"词汇并替换（如"不难发现""综上所述""毋庸置疑"等）
2. 把所有超过60字的长句拆分为短句
3. 在适当位置加入口语化表达（如"说真的""讲真""我个人觉得"）
4. 打破过于规整的段落结构，让段落长短参差不齐
5. 在2-3处加入作者的个人观点或吐槽（可以稍微有点偏激）
6. 保持原文的核心信息和逻辑不变

文章内容：
[粘贴文章]
```

### 6.3 人工二次润色（不可省略的 15 分钟）

工具润色之后，还需要人工过一遍。主要看三点：

**①加入你自己的真实经历**

哪怕一两句话也好。比如写到某个工具的时候，加一句"我第一次用这个功能的时候差点搞崩了服务器"——这种细节是 AI 编不出来的，也是读者最容易产生共鸣的地方。

**②检查数据的真实性**

AI 有时候会编造数据。凡是出现具体数字（"提升了 80%""节省了 3 小时"）的地方，都要核实一遍，或者改成更模糊但真实的表述（"明显提升""省了不少时间"）。

**③调整开头和结尾**

开头和结尾是读者印象最深的地方，也是 AI 最容易写得平庸的地方。建议人工重写，或者至少大幅改动。

## 七、质检：发布前的最后一道关

### 7.1 质检清单（逐条核对）

```
□ 标题：有没有违禁词（可用微信公众号助手检测）
□ 正文：有没有超链接失效
□ 正文：有没有错别字（重点检查同音字）
□ 数据：所有数据有没有来源支撑
□ 图片：图片有没有版权问题
□ 排版：手机端预览是否正常（PC端看着好看，手机端可能乱）
□ 违禁词：政治敏感词、医疗绝对化用语等
□ AI检测：用腾讯朱雀或类似工具跑一遍
```

### 7.2 用 OpenClaw 做自动质检

```
请对以下公众号文章进行质检，检查项目：

1. 违禁词排查：找出可能触发微信审核的敏感词
2. 事实核查：标出所有数据和事实陈述，并说明是否可信
3. 逻辑检查：找出论证不充分或自相矛盾的地方
4. 排版建议：指出可能影响阅读体验的排版问题
5. 标题评分：对当前标题打分（1-10），并给出改进建议

文章：
[粘贴文章]
```

### 7.3 违禁词处理

公众号违禁词分几类，最容易踩的是：

- **医疗类**："根治""最有效""100%有效"等绝对化表述
- **金融类**："保本""稳赚""年化收益XX%"等
- **政治类**：各类政治敏感词（AI文章一般不涉及，但要注意）
- **夸大类**："第一""最佳""唯一"等极限词

遇到这类词，直接让 OpenClaw 替换：

```
以下句子包含违禁词，请改写，保持原意但去掉违禁词：
[粘贴句子]
```

## 八、自动发布：一条指令推送草稿

质检通过之后，发布就是一条指令的事：

### 8.1 推送到草稿箱

```
将以下文章推送到微信公众号草稿箱：

标题：[文章标题]
封面图：[图片URL或描述]
摘要：[100字以内的摘要]

正文：
[粘贴完整文章]
```

OpenClaw 会自动处理排版（加粗、分段、链接等），然后推送到草稿箱。

### 8.2 封面图生成

封面图是影响点击率的关键。可以让 OpenClaw 调用图片生成工具：

```
为以下文章生成一张公众号封面图：
- 标题：[文章标题]
- 风格：简洁、科技感、深色背景
- 尺寸：900x500（公众号推荐尺寸）
- 要求：标题文字清晰可读，有品牌感
```

如果你有固定的封面模板，可以让 OpenClaw 直接套模板生成。

### 8.3 定时发布设置

推送到草稿箱后，在公众号后台选择发布时间。根据微信公众号行业普遍数据，以下时间段打开率相对较高，可作参考（具体最佳时间因受众群体不同而有差异，建议结合自己账号的历史数据判断）：

- **工作日**：早 7:30-8:30（通勤时间），午 12:00-13:00（午休），晚 20:00-22:00（睡前）
- **周末**：上午 9:00-11:00 效果相对较好

## 九、完整自动化脚本：把上面所有流程串起来

下面是一个把选题→写作→降AI味→质检→推送草稿全部串联起来的脚本思路：

```python
# wechat_auto_publish.py
# 微信公众号自动化发布流程

import os
import json
import requests
from datetime import datetime

OPENCLAW_API = "http://localhost:18789/api"

def send_to_openclaw(message):
    """向 OpenClaw 发送指令"""
    resp = requests.post(
        f"{OPENCLAW_API}/chat",
        json={"message": message},
        timeout=300
    )
    return resp.json().get("reply", "")

def step1_get_topics():
    """步骤1：获取今日热点选题"""
    with open('/tmp/hot_topics.md', 'r') as f:
        topics = f.read()
    
    prompt = f"""
    根据以下热点，推荐最适合我的公众号（AI工具/效率提升）的选题，
    给出标题和角度，格式：
    选题1：[标题] | [角度]
    选题2：[标题] | [角度]
    选题3：[标题] | [角度]
    
    热点数据：{topics}
    """
    return send_to_openclaw(prompt)

def step2_write_article(topic):
    """步骤2：写作初稿"""
    prompt = f"""
    请写一篇关于《{topic}》的公众号文章：
    - 字数：3000-4000字
    - 风格：专业实用，口语化
    - 结构：钩子开头 + 3个核心章节 + 行动号召
    - 避免AI腔调词汇
    """
    return send_to_openclaw(prompt)

def step3_reduce_ai_tone(article):
    """步骤3：降AI味"""
    prompt = f"""
    请对以下文章进行降AI味润色：
    1. 替换AI腔调词汇
    2. 拆分长句
    3. 加入口语化表达
    4. 打破均匀段落结构
    
    文章：{article}
    """
    return send_to_openclaw(prompt)

def step4_quality_check(article):
    """步骤4：质检"""
    prompt = f"""
    请对以下文章进行质检，输出JSON格式：
    {{
        "violations": ["违禁词列表"],
        "fact_issues": ["存疑的事实陈述"],
        "logic_issues": ["逻辑问题"],
        "score": 质检评分(0-100),
        "pass": true/false
    }}
    
    文章：{article}
    """
    result = send_to_openclaw(prompt)
    return json.loads(result)

def step5_push_draft(title, article):
    """步骤5：推送草稿"""
    prompt = f"""
    将以下文章推送到微信公众号草稿箱：
    标题：{title}
    正文：{article}
    """
    return send_to_openclaw(prompt)

def main(chosen_topic=None):
    today = datetime.now().strftime('%Y-%m-%d')
    
    # 步骤1：获取选题
    if not chosen_topic:
        topics = step1_get_topics()
        print(f"今日选题建议：\n{topics}")
        chosen_topic = input("请输入选择的选题标题：")
    
    print(f"\n开始创作：{chosen_topic}")
    
    # 步骤2：写作
    print("正在生成初稿...")
    draft = step2_write_article(chosen_topic)
    
    # 步骤3：降AI味
    print("正在降AI味润色...")
    polished = step3_reduce_ai_tone(draft)
    
    # 步骤4：质检
    print("正在质检...")
    qc_result = step4_quality_check(polished)
    
    if not qc_result.get('pass'):
        print(f"质检未通过，问题：{qc_result}")
        # 可以加入自动修复逻辑
        return
    
    # 步骤5：保存并推送
    output_dir = f"/data/workspace/.draft/{today}"
    os.makedirs(output_dir, exist_ok=True)
    
    filename = f"{output_dir}/{chosen_topic[:20]}.md"
    with open(filename, 'w') as f:
        f.write(f"# {chosen_topic}\n\n{polished}")
    
    print(f"文章已保存到：{filename}")
    
    push_confirm = input("是否推送到草稿箱？(y/n): ")
    if push_confirm.lower() == 'y':
        result = step5_push_draft(chosen_topic, polished)
        print(f"推送结果：{result}")

if __name__ == '__main__':
    main()
```

## 十、踩坑记录：这些问题我替你踩过了

**坑1：微信回调验证失败**
99% 是端口问题。检查顺序：服务器防火墙 → 阿里云安全组 → OpenClaw 服务是否在运行。

**坑2：草稿推送成功但格式乱掉**
微信公众号对 HTML 标签有限制，不支持所有 CSS 样式。解决办法：让 OpenClaw 用微信支持的标签生成内容，或者用 md2html 转换后再检查。

**坑3：AI写的文章被微信检测为"疑似AI生成"**
加大人工润色比例，特别是开头和结尾要人工改写。另外，加入具体的数据、案例、个人经历，这些是 AI 检测最难识别的内容。

**坑4：选题热度高但阅读量低**
热度高不等于适合你的受众。建议建立一个"选题评分模型"：热度权重 40%，与你受众的相关性权重 60%。

**坑5：OpenClaw 生成的内容超过微信字数限制**
微信公众号单篇文章最大 20000 字，一般不会超。但如果生成了大量代码块，要注意代码格式会被微信转义，建议用截图代替代码块。

**坑6：AppSecret 过期**
微信 AppSecret 不会自动过期，但如果你手动重置过，或者被微信封号，需要重新获取。建议在脚本里加入 token 刷新逻辑。

用 OpenClaw 做公众号自动化，最大的收益不是"省了多少时间"，而是**让你能把精力集中在真正有价值的事情上**——判断选题、把控方向、加入你自己的观点和经历。

机器可以写出一篇结构完整、语法正确的文章，但那个"只有你才能写出来的东西"，还是得你来。

OpenClaw 是工具，不是替代品。
